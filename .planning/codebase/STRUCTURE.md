## Directory Purposes

**app/:**
- Purpose: Main application package; contains all business logic, adapters, and pages
- Contains: Python modules for Streamlit app, adapters, configuration, logging, utilities
- Key files: `main.py` (entry point), `pages/*.py` (feature pages), `adapters/*` (plugin system)

**app/pages/:**
- Purpose: Streamlit page implementations for each user-facing feature (F1–F6)
- Contains: One Python file per page; direct UI logic with minimal extraction to modules
- Key files:
  - `home.py` — Dashboard, metrics, AI Q&A chat interface (F1)
  - `explorer.py` — Table browser, filtering, sorting, export (F2)
  - `compare.py` — Side-by-side query result comparison (F3)
  - `settings_page.py` — DB/LLM CRUD, connection testing, app defaults (F6)

**app/adapters/:**
- Purpose: Pluggable adapter system for databases and LLMs; allows runtime selection without code changes
- Contains: Abstract base classes and concrete implementations; registry for factory lookup

**app/adapters/db/:**
- Purpose: Database adapter implementations and registry
- Contains:
  - `base.py` — Abstract `DBAdapter` class defining interface (test_connection, list_tables, get_schema, run_query, dispose)
  - `mysql.py` — MySQL implementation using SQLAlchemy + pymysql; handles connection pooling, readonly mode
  - `registry.py` — Registry mapping type names to adapter classes; factory for instantiation
  - `__init__.py` — Package marker

**app/adapters/llm/:**
- Purpose: LLM adapter implementations and registry
- Contains:
  - `base.py` — Abstract `LLMAdapter` class defining interface (generate_sql, stream_text); includes shared SQL_SYSTEM_PROMPT
  - `openai_adapter.py` — OpenAI API implementation; supports custom endpoints; extra_headers support
  - `ollama_adapter.py` — Ollama local/remote implementation
  - `registry.py` — Registry mapping type names to adapter classes; factory for instantiation
  - `__init__.py` — Package marker

**app/core/:**
- Purpose: Core utilities for application-wide concerns
- Contains:
  - `config.py` — Pydantic models (DatabaseConfig, LLMConfig, AppConfig, Settings); load/save to YAML
  - `auth.py` — `streamlit_authenticator` wrapper; require_login() function
  - `runtime.py` — Adapter resolution from settings; sidebar selector UI; settings cache invalidation
  - `session.py` — Streamlit session_state helpers (chat history, recent queries, selected DB/LLM)
  - `logger.py` — Structured logging to files (queries.log, llm.log in JSONL format)
  - `sql_safety.py` — SQL validation and sanitization; blocks DML/DDL, injects LIMIT
  - `__init__.py` — Package marker

**app/utils/:**
- Purpose: Utility functions for data processing and visualization
- Contains:
  - `schema.py` — Schema summarization for LLM context; SQL extraction from LLM responses
  - `export.py` — CSV/Excel export functions
  - `viz.py` — Auto-chart generation heuristics (bar, line, scatter based on data types)
  - `__init__.py` — Package marker

**config/:**
- Purpose: Store YAML-based configuration files
- Contains:
  - `settings.yaml` — Database and LLM configurations (generated/edited via UI, in .gitignore)
  - `settings.example.yaml` — Template with examples (MySQL, OpenAI, Ollama)
  - `auth.yaml` — User credentials with bcrypt hashes and cookie config
- Generated by: Settings UI (settings.yaml), provided manually (auth.yaml)

**logs/:**
- Purpose: Application log files
- Contains:
  - `queries.log` — Every executed query (JSONL): timestamp, user, database, SQL, rows, duration, errors
  - `llm.log` — Every LLM call (JSONL): timestamp, user, model, question, generated SQL, duration, errors
- Format: JSON Lines (one JSON object per line) for easy parsing/analysis
- Lifecycle: Auto-created if missing; manually deleted for cleanup

## Naming Conventions

**Files:**
- Python modules: lowercase with underscores (e.g., `config.py`, `sql_safety.py`)
- Pages: descriptive lowercase (e.g., `home.py`, `explorer.py`, `settings_page.py`)
- Adapter implementations: `<provider>_adapter.py` or `<provider>.py` (e.g., `openai_adapter.py`, `mysql.py`)
- Log files: descriptive lowercase with `.log` suffix (e.g., `queries.log`, `llm.log`)

**Directories:**
- Package directories: lowercase plural for collections (e.g., `pages/`, `adapters/`, `utils/`, `core/`)
- Sub-packages: grouped by concern (e.g., `adapters/db/`, `adapters/llm/`)

**Classes:**
- Adapters: `<Provider>Adapter` (e.g., `MySQLAdapter`, `OpenAIAdapter`, `OllamaAdapter`)
- Models/Configs: `<Domain>Config` (e.g., `DatabaseConfig`, `LLMConfig`)
- Data classes: `<Domain>Result` (e.g., `SafetyResult`)

**Functions:**
- Private (module-level): prefixed with `_` (e.g., `_auth_path()`, `_make_logger()`)
- Public API: lowercase with underscores (e.g., `require_login()`, `load_settings()`, `validate_and_sanitize()`)

**Constants:**
- Uppercase with underscores (e.g., `_ALLOWED_STATEMENT_TYPES`, `_FORBIDDEN`, `SQL_SYSTEM_PROMPT`)
- Session state keys: uppercase with leading underscore (e.g., `_CHAT_HISTORY_KEY`, `_SELECTED_DB_KEY`)

## Where to Add New Code

**New Database Adapter (e.g., PostgreSQL):**
1. Create `app/adapters/db/postgres.py` — inherit from `app.adapters.db.base.DBAdapter`
2. Implement required methods: `test_connection()`, `list_tables()`, `get_schema()`, `run_query()`, optionally `dispose()`
3. Register in `app/adapters/db/registry.py` — add `"postgres": PostgresAdapter` to `_REGISTRY`
4. Adapter becomes available in Settings UI immediately

**New LLM Adapter (e.g., Anthropic):**
1. Create `app/adapters/llm/anthropic_adapter.py` — inherit from `app.adapters.llm.base.LLMAdapter`
2. Implement required methods: `generate_sql()`, `stream_text()`
3. Register in `app/adapters/llm/registry.py` — add `"anthropic": AnthropicAdapter` to `_REGISTRY`
4. Adapter becomes available in Settings UI immediately

**New Page / Feature:**
1. Create `app/pages/<feature_name>.py` following Streamlit conventions
2. At minimum: call `st.title()`, handle authentication via `require_login()` (called in main.py), use `resolve_selected_db()` and `resolve_selected_llm()` if needed
3. Register in `app/main.py` — add `st.Page(str(pages_dir / "<feature_name>.py"), title="...", icon="...")` to navigation dict

**Utility Functions:**
- Data processing: Add to `app/utils/schema.py` (schema-related) or create new `app/utils/<domain>.py`
- Exports: Add to `app/utils/export.py`
- Visualization: Add to `app/utils/viz.py`
- Core concerns: Add to appropriate `app/core/<domain>.py` file

**Adding Configuration:**
- New DB/LLM config fields: Update Pydantic models in `app/core/config.py` (DatabaseConfig, LLMConfig, or AppConfig)
- Default values set via `Field(default=...)` in Pydantic
- Settings UI in `app/pages/settings_page.py` auto-reflects new fields via form generation

## Special Directories

**logs/:**
- Purpose: Application log files
- Generated: Yes (auto-created if missing)
- Committed: No (in .gitignore)
- Format: JSONL (one JSON object per line); human-readable, machine-parseable
- Manual cleanup: Delete files if space becomes an issue

**config/:**
- Purpose: Configuration files
- Generated: `settings.yaml` (auto-generated by Settings UI); `auth.yaml` (manual, must be created)
- Committed: `settings.example.yaml` yes, `settings.yaml` no (in .gitignore), `auth.yaml` no (in .gitignore)
- Usage: YAML loaded via `app/core/config.py` (settings) and `app/core/auth.py` (auth)

**.env:**
- Purpose: Environment variables for secrets and overrides
- Generated: No (copy from `.env.example` manually)
- Committed: No (in .gitignore)
- Contents: OPENAI_API_KEY, AUTH_PATH override, SETTINGS_PATH override, LOG_DIR override
- Never read by any GSD tool; environment-only

**.planning/:**
- Purpose: GSD planning documents (architecture, structure, testing, conventions, concerns)
- Generated: Yes (by `/gsd-map-codebase` and related commands)
- Committed: Yes (provides reference for future implementation phases)
- Not code; documentation for developer guidance
