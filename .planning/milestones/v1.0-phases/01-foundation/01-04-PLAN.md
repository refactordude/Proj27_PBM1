<objective>
Enforce AGENT-08 by wiring a 30-second `httpx.Timeout` onto every `chat.completions.create` invocation in `app/adapters/llm/openai_adapter.py`. Two call sites exist today — `generate_sql` (line ~50) and `stream_text` (line ~61). Both get the timeout via a module-level constant `_REQUEST_TIMEOUT = httpx.Timeout(30.0)` so the value is DRY and unit-testable.

Purpose: Prevent production hangs when OpenAI's API is slow or unresponsive.     class RequestTimeoutConstantTest(unittest.TestCase):
        def test_timeout_is_httpx_timeout_30s(self) -> None:
            self.assertIsInstance(_REQUEST_TIMEOUT, httpx.Timeout)

            adapter = _mk_adapter()

            self.assertIn("timeout", kwargs)

        unittest.main()
